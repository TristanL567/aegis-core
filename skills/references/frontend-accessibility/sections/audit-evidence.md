# Audit Evidence

- Automated scanners are useful for missing names, invalid ARIA, contrast,
  labels, landmarks, and structural issues, but they do not prove keyboard
  usability, focus quality, comprehension, or screen reader flow.
- Manual review should cover keyboard-only operation, visible focus, reading
  order, names and descriptions, form errors, dynamic updates, zoom, contrast,
  motion preferences, and critical user paths.
- Assistive technology evidence should name the browser, platform, assistive
  technology, interaction path, and observed behavior when those details matter.
- Findings should identify the affected element or flow, user impact,
  reproducible steps, expected behavior, and evidence source.
- Severity should account for task criticality, frequency, affected users,
  workaround availability, legal or policy requirements, and whether the issue
  blocks completion.
- Passing evidence should be scoped to what was actually checked rather than
  treated as blanket conformance.
