# Visual And Responsive Accessibility

- Text, icons, borders, focus states, and meaningful graphical elements need
  sufficient contrast against their backgrounds.
- Text should remain usable with browser zoom, text scaling, increased spacing,
  and narrow viewport widths without clipping or overlapping critical controls.
- Target size and spacing should support touch and motor accessibility,
  especially for repeated actions, destructive actions, and dense toolbars.
- Color should not be the only indicator of state, requiredness, error,
  selection, or data meaning.
- Motion, animation, parallax, and auto-updating content should respect reduced
  motion preferences when movement is nonessential or vestibularly risky.
- Responsive layouts should preserve logical order, visible labels, focus order,
  and access to all controls across breakpoints.
