# External Cross-References

## Purpose

This folder records external repos, tools, concepts, and source links that may
influence AEGIS. Each entry explains what was referenced, when it was retrieved,
why it is relevant, and how it may be used without changing AEGIS authority.

## Rules

- External references are evidence and inspiration, not canonical AEGIS
  behavior.
- `AEGIS.md`, `contracts/`, `skills/`, and `execution/` remain authoritative
  for ticketing, roles, validators, conformance, and completion reporting.
- Every external reference must name source URLs, retrieval date, relevance,
  risks, and allowed integration paths.
- Generated artifacts from external tools must not be committed unless a ticket
  explicitly allows them.
